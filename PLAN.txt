PASSWORD MANAGER BACKEND PLAN
STACK
- FastAPI
- PostgreSQL
- AWS RDS
- psycopg2
- bcrypt for password hashing
- AES/Fernet for encryption

--------------------------------------------------

PHASE 1 — USER ACCOUNTS (AUTHENTICATION)

Goal:
Allow users to register and log in securely.

Database Table: users

fields
- id (UUID, primary key)
- first_name
- last_name
- email (unique)
- password_hash
- created_at

Features
- Register account
- Login
- Hash passwords
- Verify passwords
- JWT authentication
- Protect routes

Tasks
1. Create users table
2. Build register API
3. Hash password with bcrypt
4. Store user in database
5. Build login API
6. Verify password
7. Generate JWT token

--------------------------------------------------

PHASE 2 — VAULT SYSTEM

Goal:
Each user has a vault where passwords are stored.

Database Table: vaults

fields
- id
- user_id
- vault_name
- created_at

Features
- Create vault
- List vaults
- Delete vault

Example vaults
- Personal
- Work
- Crypto

--------------------------------------------------

PHASE 3 — PASSWORD STORAGE

Goal:
Store website credentials.

Database Table: password_entries

fields
- id
- vault_id
- title
- username
- password_encrypted
- website
- notes
- created_at
- updated_at

Example entry

Title: Gmail
Username: user@gmail.com
Password: encrypted
Website: gmail.com

IMPORTANT
Never store raw passwords.
Passwords must be encrypted.

--------------------------------------------------

PHASE 4 — ENCRYPTION SYSTEM

Goal:
Encrypt stored passwords.

Workflow

User logs in
↓
Encryption key derived
↓
Passwords encrypted before storing

Use
- cryptography library
- Fernet encryption

Database stores
password_encrypted
